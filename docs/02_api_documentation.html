1 Overview and Conventions

The ComplianceOS backend exposes a RESTful JSON API. All endpoints are versioned under the /api/v1 prefix.

Base URL: https://<render-service>.onrender.com/api/v1

Required Request Headers

All endpoints (except POST /auth/login) require the following headers on every request:

HeaderDescriptionExample
X-User-IdThe authenticated user's unique identifieruser-admin
X-RoleThe authenticated user's roleadmin

Role Badges

Each endpoint indicates the minimum required role using these badges:

All Roles Any authenticated user may call this endpoint.
Admin Requires the admin role.
Owner Requires admin or owner role.
Auditor Requires admin or auditor role.

Common Response Codes

CodeMeaning
200 OKSuccessful GET or PATCH
201 CreatedNot used; successful POST returns 200 with the created resource
400 Bad RequestValidation error or invalid field values
401 UnauthorizedMissing authentication headers
403 ForbiddenAuthenticated but insufficient role
404 Not FoundRequested resource does not exist
500 Internal Server ErrorDatabase or unexpected server error

2 Authentication

POST /auth/login

Authenticates a user by role and returns the session payload to be stored client-side. No headers required.

Request Body

{ "role": "admin" }

Accepted values for role: admin, owner, auditor

Response

{ "userId": "user-admin", "name": "Priya Sharma", "role": "admin" }

4 Risks

GET /risks All Roles

Returns all risks. Optional query parameter: status (e.g., ?status=Open).

POST /risks Owner

Creates a new risk. The score field is auto-calculated by the database as likelihood * impact.

Request Body

{ "title": "string", "description": "string", "likelihood": 4, "impact": 5 }
GET /risks/{risk_id} All Roles

Returns a single risk with its full list of linked controls.

PATCH /risks/{risk_id} All Roles

Partially updates a risk. Accepts any subset of title, description, likelihood, impact, status.

Accepted status values: Open, Mitigated, Accepted

POST /risks/{risk_id}/link-control All Roles

Links an existing control to a risk.

Request Body

{ "control_id": "C01" }
DELETE /risks/{risk_id}/link-control/{control_id} All Roles

Removes a control link from a risk.

5 Controls

GET /controls All Roles

Returns all controls including their dependency relationships. Optional query parameters: status, owner_id, trust_criteria.

GET /controls/{control_id} All Roles

Returns a single control with its linked risks, evidence artifacts, and dependency relationships.

PATCH /controls/{control_id} Owner

Updates a control's status or owner_id.

Accepted status values: Not Started, In Progress, Implemented, Verified

POST /controls/{control_id}/link-dependency Owner

Creates a typed dependency relationship between two controls.

Request Body

{ "related_control_id": "C02", "relationship": "supplements" }

Accepted relationship values: supplements, depends_on, replaces

DELETE /controls/{control_id}/link-dependency/{related_control_id} Owner

Removes a control dependency relationship.

6 Evidence

GET /evidence All Roles

Returns all evidence artifacts. Optional query parameters: status, control_id.

POST /evidence/controls/{control_id}/evidence All Roles

Uploads an evidence artifact for a specific control. Accepts multipart/form-data. Provide either a file field (binary upload) or a link_or_text field (URL or text reference). Newly uploaded evidence defaults to Pending Review status.

PATCH /evidence/{evidence_id}/review Auditor

Sets the review outcome for an evidence artifact. Requires Admin or Auditor role.

Request Body

{ "status": "Rejected", "rejection_reason": "Scan report is from a development environment." }

Accepted status values: Approved, Rejected, Needs Resubmission

The rejection_reason field is stored and surfaced in all subsequent responses.

7 DPDP

GET /dpdp/requests All Roles

Returns all data principal requests ordered by ascending SLA due date (most urgent first).

POST /dpdp/requests Admin

Logs a new data principal request. The SLA due date is automatically calculated as received_on + 30 days in accordance with DPDP Act timelines. Status defaults to Open.

Request Body

{ "request_type": "Erasure", "data_principal_name": "Ananya Rao", "received_on": "2026-07-01" }

Accepted request_type values: Access, Erasure, Correction

PATCH /dpdp/requests/{request_id} Admin

Updates a request's status, assignee, or rejection reason. Auditors cannot be assigned to Erasure type requests. When setting status to Rejected, the rejection_reason field must be provided.

Request Body

{ "status": "Rejected", "assigned_to": "user-admin", "rejection_reason": "Request falls outside DPDP exemption criteria." }

Accepted status values: Open, In Progress, Completed, Rejected

GET /dpdp/consent-log All Roles

Returns all consent log entries ordered by descending timestamp.

POST /dpdp/consent-log Admin

Records a new consent declaration for a data principal.

Request Body

{ "data_principal_name": "Sneha Reddy", "purpose": "Newsletter Subscription Emails", "consent_status": "Given" }

Accepted consent_status values: Given, Withdrawn, Pending

8 Activity Log

GET /activity-log All Roles

Returns system activity log entries in descending chronological order. This endpoint is read-only. Entries are written automatically by the backend on every state-changing operation.

Optional query parameters:

ParameterTypeDescription
entity_typestringFilter by module, e.g. Risk, Evidence, DPDP Request
entity_idstringFilter by a specific record ID
limitintegerMaximum number of results, defaults to 50
